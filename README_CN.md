# Stock Target Price Skill — AI Agent 股票估值分析技能

一个面向 AI Agent 的结构化股票估值提示词框架，内置**自适应搜索循环**、**动态信息图谱**、**三波研究节奏**，以及对每个关键判断的**显式推演卡片**，同时支持按需启用的宏观增强模块。通过"探索 → 聚焦 → 反证"三波研究搭建经营事实底座，判断是否需要"宏观—行业—公司"三层推演，完成标准化盈利与现金流处理，识别关键价值驱动因素，选择匹配的估值方法，并通过情景分析、敏感性测试与反证框架校验结论。

## 功能概述

触发后，该技能会引导 AI Agent 完成一套严谨的 **13 步估值流程**（第 0–12 步），并在必要时激活**宏观增强模块（M1–M4）**。四个贯穿全流程的机制——**自适应搜索循环**、**信息图谱**、**研究波次**、**推演卡片**——用于防止估值退化为对历史的线性外推。

### 迭代研究方法论

框架把估值视为迭代推演，而不是"一次性搜索 + 按模板填空"。

**1. 自适应搜索循环** —— 每次信息获取（搜索、文档阅读、数据查询）须完成一次循环，杜绝盲目搜索：

- **搜索前**：声明要回答的具体问题、预期信号（证实 / 证伪 / 意外）、在信息图谱中的位置
- **搜索后**：评估回答度、将新内容分层（事实 / 推断 / 假设）、捕获意外信号、更新图谱
- **下一步决策**（五选一）：推进 / 深挖 / 转向 / 回退 / 标记缺口

同一问题连续 2 次搜索无增量须触发诊断暂停——不允许空转循环。

**2. 信息图谱** —— 动态维护的活地图，分四层：已确认事实 / 活跃假设 / 信息缺口 / 意外发现。图谱驱动每次"下一步决策"，也决定每波何时达到停止条件。

**3. 三波研究** —— 信息抓取分波次循环，每波目标与停止条件明确：

| 波次 | 目标 | 在流程中的主要落点 |
|------|------|------------------|
| **Wave 1 · 建盘** | 广度优先的事实底稿——业务构成、财务节奏、行业位置、宏观暴露 | 第 0–3 步 |
| **Wave 2 · 聚焦** | 围绕候选核心变量的定向深抓，收敛至 1–3 个真正的驱动变量 | 第 4 步 |
| **Wave 3 · 反证** | 主动寻找可能推翻 base case 的证据 | 第 10 与 11 步之间 |

Wave 2/3 发现新关键变量或证伪已锁定变量时，分析须显式回到第 4 步更新，并在报告中记录回退过程——不得隐藏。

**4. 推演卡片** —— 关键判断以结构化卡片呈现（全流程通常 5–7 张，不堆砌）：

```
结论 → 前提事实（来源 / 时点 / 层级：事实|推断|假设）
    → 推理链（每一跳标注逻辑类型）
    → 每跳置信度（显式指出最脆弱一跳）
    → 反证条件
    → 信息缺口
```

卡片仅在三类节点构建：每个核心变量（第 4 步）、每个关键估值参数（第 8 步）、每个情景的假设组合（第 10 步）。卡片是活的——新证据可调整置信度，反证可触发回退。

### 核心分析流程

| 步骤 | 内容 |
|------|------|
| 第 0 步 | 界定估值对象、会计口径与宏观暴露 |
| 第 1 步 | 搭建历史经营事实底座（利润表、资产负债表、现金流） |
| 第 2 步 | 标准化盈利与现金流（剥离一次性项目及宏观扰动） |
| 第 3 步 | 理解商业模式、行业结构、管理层与宏观传导机制 |
| 第 4 步 | **锁定 1–3 个核心变量并完成 Wave 2 定向验证**（子问题、信息源、领先指标、抓取后动作） |
| 第 5 步 | 建立"分业务 + 分驱动 + 分情景"的预测框架 |
| 第 6 步 | 用资本效率、现金流质量与资产负债表韧性校验预测 |
| 第 7 步 | 选择与商业模式匹配的估值方法 |
| 第 8 步 | 基于质量、周期、风险溢价与隐含预期确定估值参数 |
| 第 9 步 | 将企业价值桥接到每股股权价值 |
| 第 10 步 | 三情景分析（宏观路径 × 公司执行矩阵）与敏感性测试 |
| 第 11 步 | 反向预期校验、Wave 3 反证清单处理、与市场一致预期对比 |
| 第 12 步 | 收敛为可解释、可跟踪、可更新的估值判断 |

### 宏观增强模块（按需激活）

当公司价值显著受利率、汇率、商品价格、关税、地缘政治等外生变量驱动时，自动激活：

| 步骤 | 内容 |
|------|------|
| M1 | 用 3–6 个关键外生变量定义当前宏观 regime |
| M2 | 构建 Bear / Base / Bull 三条宏观情景路径 |
| M3 | 将宏观变量映射到公司层具体传导路径 |
| M4 | 定义监控面板——领先指标与 base case 失效条件 |

所有宏观启用时对主流程的增补要求（标准化多拆一层、压力测试、参数一致性、反向校验中的隐含路径等）**统一集中在模块内一张表格**，主流程保持"宏观中性"的清晰叙述，不再散落 if/else 条件分支。

输出以**估值结论摘要**（目标价、三情景、核心假设、宏观路径）开头，随后展开叙述型分析报告——关键判断的前提、推理链与反证条件须在叙述中自然呈现，不得被"因此""综合来看"等结论性表述掩盖。

## 行业覆盖

该技能内置了针对不同行业的分析框架，每个行业均包含宏观补充维度：

- **制造业 / 汽车零部件 / 工业** — 出货量、ASP、产能利用率、资本开支 + 宏观：制造业投资、关税、汇率、原材料
- **消费** — 同店增长、渠道扩张、提价能力、库存 + 宏观：居民收入、就业、消费信心、财富效应
- **互联网 / 软件 / 平台** — 用户增长、ARPU、股票薪酬、经营杠杆 + 宏观：广告预算、IT 支出、融资环境、实际利率
- **半导体 / 硬件** — ASP、良率、利用率、周期位置 + 宏观：全球电子需求、AI capex、出口限制
- **银行 / 保险 / 券商** — ROE、资产质量、拨备、资本充足率 + 宏观：利率曲线、信用成本、监管导向
- **周期 / 资源 / 化工 / 航运** — 价格周期、供给约束、成本曲线 + 宏观：全球需求、能源、美元、贸易政策
- **未盈利生物科技 / 早期硬科技** — 里程碑概率、现金消耗、事件敏感性 + 宏观：融资环境、利率、监管审批

## 安装方法

### 快速安装（推荐）

如果你的 AI Agent 支持安装技能（如 Claude Code、OpenClaw 等），直接告诉它：

```
帮我安装这个股票估值技能 https://github.com/non-convex/stock-target-price-skill
```

Agent 会自动下载并配置该技能。

### 手动安装

```bash
git clone https://github.com/non-convex/stock-target-price-skill.git

# 以 Claude Code 为例：
mkdir -p ~/.claude/skills/stock_target_price
cp stock-target-price-skill/SKILL.md ~/.claude/skills/stock_target_price/SKILL.md
cp -R stock-target-price-skill/references ~/.claude/skills/stock_target_price/references
```

### 作为独立提示词使用

本项目现在采用**模块化提示词结构**：`SKILL.md` 作为入口，详细规则按需从 `references/` 读取。你可以：

1. **作为 skill 入口**：先加载 `SKILL.md`，再让 Agent 按需打开被引用的模块文件。
2. **作为系统提示词**：如果你的环境不能按需读取本地文件，可将 `SKILL.md` 与 `references/dynamic-research.md`、`references/core-workflow.md`、`references/output-format.md` 拼接使用；只有在需要时再追加 `references/macro-module.md` 或 `references/industry-playbooks.md`。
3. **作为工具/技能**：把整个文件夹集成到 Agent 框架（如 LangChain、AutoGPT、自定义 Agent）中，让入口提示词逐步加载详细模块。
4. **作为参考框架**：将 13 步流程与动态研究循环一起作为你自己估值工作的检查清单。

## 输出格式

分析输出遵循以下结构：

1. **估值结论**（置顶） — 目标价、三情景区间、可选的概率加权目标价、核心假设、宏观路径摘要、上行/下行空间、催化剂与风险、跟踪指标、免责声明
2. **叙述型分析报告**（10 个板块，若宏观启用则 + M1–M4 路径） — 从事实底座到最终判断的完整推导；关键判断的前提、推理链与反证条件在叙述中自然呈现

## 自定义

你可以分层修改这个 skill：

- `SKILL.md` —— 主控逻辑、导航方式、高优先级约束
- `references/dynamic-research.md` —— 自适应动态搜索、信息图谱、研究波次、推演卡片
- `references/core-workflow.md` —— 估值流程步骤
- `references/macro-module.md` —— 宏观触发条件与增补逻辑
- `references/industry-playbooks.md` —— 行业适配规则
- `references/output-format.md` —— 输出模板与防错清单

## 环境要求

- 具备联网搜索能力的 AI Agent（用于获取财务数据）
- 大上下文 LLM（建议 128K+ tokens 以获得完整分析）
