# 股票估值分析（入口）

你要为**单只股票**给出独立、可审计、可回退的目标价分析。
本 skill 采用**模块化结构**：`SKILL.md` 负责主控与导航，详细规则按需读取 `references/` 下的文件。

## 开始前先读哪些文件

开始分析前，按下面顺序加载文件：

1. `references/dynamic-research.md` —— **必读**。定义当前的自适应动态搜索、信息图谱、三波研究与推演卡片。
2. `references/core-workflow.md` —— **必读**。定义 Step 0–12 的主流程、每步必做动作与产出。

仅在需要时追加读取：

3. `references/macro-module.md` —— 当外生变量显著驱动公司价值时读取。
4. `references/industry-playbooks.md` —— 只读取与目标公司最相关的行业章节。
5. `references/output-format.md` —— 输出前必读；沿用当前输出格式与防错要求。

## 主控规则

- **Step 是唯一主轴**：始终按 Step 0–12 推进；Wave 只是当前研究姿态，不另起一套顺序。
- **动态搜索优先于模板填空**：每次搜索、读文档、查数据，都必须执行一次自适应动态搜索闭环。
- **事实先于估值**：先建立事实底座，再标准化盈利与现金流，再选择估值方法与参数。
- **核心变量必须收敛**：全流程只锁定 1–3 个真正驱动价值的变量；被新证据击穿时立即回到 Step 4。
- **关键判断必须可回溯**：任何重要结论都要能回到前提事实、推理链、最脆弱一跳和反证条件。
- **缺口必须显式记录**：证据不足时标记信息缺口，不得靠叙述补足证据。

## 自适应动态搜索（全程默认）

自适应动态搜索是本 skill 的主控逻辑，不是附加说明。每次信息获取都执行下列最小闭环：

1. **搜索前**：写清本次要回答的具体问题、预期证实/证伪/意外信号、它在信息图谱中的位置。
2. **搜索后**：判断回答度；把新信息分类为事实/推断/假设；更新图谱；记录意外发现。
3. **下一步决策**：只能在“推进 / 深挖 / 转向 / 回退 / 标记缺口”五项中选择一项。
4. **空转保护**：同一问题连续 2 次无增量，先诊断搜索意图、关键词和数据源；仍无增量则标记缺口，不再空转。
5. **动态回退**：Wave 2 或 Wave 3 若发现新关键变量，或推翻已锁定变量，立即回退到 Step 4 并记录触发原因。

## 执行顺序

- 先按 `references/dynamic-research.md` 建立图谱、波次与推演卡片约束。
- 再按 `references/core-workflow.md` 执行 Step 0–12。
- 满足宏观触发条件时，插入 `references/macro-module.md` 的 M1–M4。
- 需要行业细则时，只读取 `references/industry-playbooks.md` 中对应章节。
- 准备输出时，读取 `references/output-format.md`，并严格沿用当前输出结构。

## 不要做的事

- 不要把当前股价当作估值输入；它只用于反向校验。
- 不要绕过 Wave 2 直接进入预测。
- 不要把订单额、定点金额、意向协议直接当收入。
- 不要让行业叙事替代收入、利润率、现金流、股本这四个硬变量。
- 不要在景气高点利润上再叠加高倍数。
